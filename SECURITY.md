# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository, please report it privately.

- **Email:** support@sparkyventurelabs.com
- **Preferred:** open a [private security advisory](../../security/advisories/new) on this repo.

Please do **not** open a public issue for security problems.

## What to include

- A description of the issue and its impact.
- Steps to reproduce, or a proof of concept.
- Any relevant logs, versions, or configuration (with secrets redacted).

## Response

We aim to acknowledge reports within a few business days and to keep you updated as we
investigate and resolve the issue.

## Scope

This policy covers the code and configuration in this repository. Secrets, API tokens, and
signing keys are never stored in git — if you believe one has been exposed, report it
immediately using the contact above.
